surtr

Simple tool comics library synchronization

Download this project as a .zip file Download this project as a tar.gz file

What is surtr?

surtr is a tool that allows to synchronize all or some comics of a library with another folder.

Why surtr?

About the name, please have fun reading this.

For a long time, I had difficulties to synchronize my comcis on my tablet. At first, my tablet could contain all my comics. So standard sync tools were enough. Then came a moment where my tablet was not big enough to contain all my library. I looked for a tool allowing me to sync a part of the library. But unfortunately, I have never found one. This kind of tool exist for music. Playlists are a common thing for music library tools. This is exactly what I needed, but nothing for standard files.

So the idea of surtr was born. Being able to create playlists for a comics library, and sync one or more playlists to my tablet, with a criteria of maximum size and alerts when this maximum was exceeded.

surtr does not provide, yet, this feature. But it's still usable. What it does for now is allow to set comics as favorite, and these favorites will be sync'd. This is all file based, and all files within a folder (and sub-folders) will be included in the library. Supported formats are zip/cbz, rar/cbr and pdf.

For now, what I do is sync the favorite files to a shared folder (on a NAS for example) and sync this folder with my tablet. This is done by tools like the excellent FolderSync (free version available, but the developer deserve to be paid for his work).

surtr also provides really simple features, like opening the comic with an external viewer, and delete the file from both the library and the computer.

Beware, the UI is really, I mean REALLY, basic. It's mostly keyboard shortcuts, big ugly buttons, and it's not even safe, like for example buttons that won't be disable while an operation is running...

What is the plan?

In the future, I would like surtr being able to do more things, like:

  • normalize the extension of the files
  • convert other formats to zip/cbz
  • display the first picture of the file
  • easily edit the cover (cropping)
  • choose another page as cover
  • renaming of the files

One of the most important feature I would like, is MTP synchronization. Because my tablets have always been on Android, and I don't want to change. But Google, for a good reason, has abandoned Universel Mass Storage protocol. But they chose a really crappy protocol to replace it: MTP/PTP (Media Transfer Protocol and Picture Transfer Protocol). This protocol is a hell, is not stable, is buggy, and is not even active (stable version came with Windows Vista, no more news since). If you have issues with accessing the content of your tablet, you know why now.

I would like to improve the UI too. I won't work on a theme system or shiny stuff, but the current UI is ugly.

Also, on a more technical aspect, I didn't really use TDD. Which is bad. So I'll improve this aspect in the future.

Something else?

And I don't really like this choice (I respect it though, it's their choice). Prism is not what I call an easy framework and it's far from perfect. Especially since I've been using a far more performant framework at work (base on Prism). Unfortunately it's proprietary. And I will be able to put Prism on my CV ;)

That's about it. If you want to try this tool, be my guest. But it's not yet packaged as a usable tool. It will be, if some are insterested (I don't expect that though).
